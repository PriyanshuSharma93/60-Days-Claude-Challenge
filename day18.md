# 🎯 Objective

The objective of Day 18 was to build a reusable **Brain Dump Action Planner** using Claude AI that transforms unstructured information into a structured, interactive project dashboard.

The challenge focused on designing an AI workflow capable of organizing meeting notes, brainstorming sessions, voice memo transcripts, class notes, and project discussions into actionable insights without inventing or assuming missing information.

# 📚 What I Learned

## 1. Building a Reusable AI Productivity Skill

Instead of creating one-time prompts, I developed a reusable AI skill that consistently transforms raw notes into organized project dashboards.

The skill supports multiple workflows, including:

* Full Breakdown
* Transcript Mode
* Merge Mode

This demonstrated how prompt engineering can automate repetitive documentation tasks.

---

## 2. Information Organization

The generated dashboard automatically organizes content into:

* Summary
* Key Takeaways
* Action Items
* Open Questions
* Risks & Blockers
* Conflicts
* Additional Notes

Everything is extracted directly from the provided notes without adding assumptions.

---

## 3. Interactive Dashboard Design

The generated HTML dashboard includes:

* Responsive Layout
* Dashboard Cards
* Interactive Tables
* Status Badges
* Collapsible Sections
* Clean Typography
* Modern UI Design

The design is inspired by productivity platforms such as Notion, ClickUp, Linear, Asana, and Airtable.

---

## 4. Transcript & Merge Processing

The skill supports advanced note processing by generating:

### Transcript Mode

* Speaker Summary
* Decisions by Speaker
* Action Items by Speaker
* Attribution Notes

### Merge Mode

* Duplicate Items
* Conflict Review
* Source Information

This makes it useful for meetings, workshops, and collaborative projects.

---

## 5. Reliable AI Output

A key principle of this skill is maintaining accuracy.

The workflow:

* Never invents missing information
* Never assumes owners or deadlines
* Preserves names, dates, and terminology exactly as provided
* Clearly marks missing fields as **"Not specified"**

This ensures trustworthy and transparent outputs.
