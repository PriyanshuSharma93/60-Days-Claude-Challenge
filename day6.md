# 📚 What I Learned

## 1. What is an ATS?

An **Applicant Tracking System (ATS)** is software used by companies to scan, filter, and rank resumes before they reach a recruiter.

Instead of reading every resume manually, recruiters rely on ATS to identify candidates whose resumes match the job description.

---

## 2. Why ATS Optimization Matters

A technically strong candidate can still be rejected if the resume:

* Uses poor formatting
* Lacks relevant keywords
* Is difficult for ATS to parse
* Contains unnecessary graphics or tables
* Doesn't clearly highlight technical skills and projects

Optimizing a resume increases the chances of passing the ATS screening stage.

# 💡 ATS Best Practices

I learned that an ATS-friendly resume should:

* Use a single-column layout
* Avoid tables, icons, images, and text boxes
* Use clear section headings
* Include relevant technical keywords
* Highlight measurable achievements where applicable
* Keep project descriptions concise
* Use strong action verbs
* Fit on a single A4 page

---

# 🚀 Practical Example

### ❌ Weak Resume Bullet

```text
Worked on a Task Manager project.
```

### ✅ Optimized Resume Bullet

```text
Developed a Task Manager application using the MERN stack with CRUD functionality, user authentication, and responsive UI, improving project organization and usability.
```

The second version is more descriptive, keyword-rich, and ATS-friendly while remaining truthful.
